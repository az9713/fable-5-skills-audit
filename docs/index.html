Skills Audit · a reduction ledger

Case study · orchestrated with Fable 5

A Claude Code install nobody could hold in their head, cut to something you can.

One personal configuration had grown to 182 skills across 712 files and 1.5 GB, plus 67 plugins loading into every session. A four-pass audit — orchestrated on Fable 5, fanning out 17 parallel readers — took it apart and put back only what earns its place.

182 → 98 personal skills kept active — 84 archived or merged, every one reversible

03

The four-pass method

Facts before opinions; read the real bodies, not just the descriptions; and reconcile the two sides against each other before changing anything.

Pass 0 · Inventory

Count everything, trust nothing

Script-walk every skill and plugin: size, sub-skills, last-modified, broken references, malformed frontmatter → CSV.

Pass 1 · Categorize

Make it legible

Sort all 182 skills into 20 domains so overlaps become comparable. Zero left uncategorized.

Pass 2 · Grade overlaps

Read the bodies in parallel

17 subagents grade every pair Duplicate / Subset / Overlapping / Adjacent, cross-checking 334 plugin skills for collisions.

Pass 3 · Verdict & reconcile

Close the loop before cutting

A skill may be redundant because a plugin covers it — while that plugin is itself a cut candidate. Auditing both together catches the gap.

04

Two findings that paid for the whole exercise

The kind of thing that only surfaces when something actually reads the estate end to end.

Orphaned · still running

A 20-plugin vendor suite whose source was deleted months ago

Still loading from frozen cache: ~180 skills, agents and commands plus 5 live MCP connections into every session — two of them dead without paid data subscriptions the user didn’t hold. Nobody had noticed.

Triple-enabled

One design skill, installed three times

Enabled from 3 different marketplaces — hash-verified byte-identical — so a single skill name resolved three ways at once.

Reported in aggregate. Every figure here regenerates from a CSV produced by the tooling in this repository; no individual skill or plugin from the audited system is named. The method runs read-only on your own ~/.claude.

Result in one line: a 182-skill / 67-plugin estate went to 98 skills and 25 plugins, shedding ~200 items of per-session context and ~1.5 GB — with a documented undo for every change.
